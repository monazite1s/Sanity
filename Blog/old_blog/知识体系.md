## **6. 前端工程化**

### **6.1 构建工具**

#### **6.1.1 Webpack**

- **基础概念**
  - Entry, Output, Loaders, Plugins, Mode (development/production)
  - Chunk, Module, Bundle
- **核心原理**
  - AST 解析与模块依赖分析
  - Tapable 事件流机制
  - Compiler & Compilation 生命周期
  - HMR (Hot Module Replacement) 热更新实现
- **性能优化**
  - Tree Shaking & Scope Hoisting
  - Code Splitting (按需加载)
  - Cache: Babel Cache、HardSourcePlugin、Persistent Cache
  - 多进程构建：`thread-loader`
  - Webpack 5 内置优化：Module Federation

#### **6.1.2 Vite**

- **基础概念**
  - Esbuild 预构建
  - Rollup 生产构建
  - ESM + HTTP 模块导入
- **核心原理**
  - 即时服务器解析 (Dev Server)
  - HMR 基于 Native ESM
  - 预构建机制 (Esbuild)
  - Rollup 打包策略
- **优化策略**
  - SSR 方案
  - Vite + Vue3 + Pinia 生态优化
  - `optimizeDeps` 依赖预构建
  - `define` 宏替换

#### **6.1.3 Rollup**

- **基础概念**
  - Tree Shaking & Dead Code Elimination
  - Output Format (ESM, CJS, IIFE, UMD)
- **核心原理**
  - AST 解析与代码转换
  - Plugin 机制与生命周期
  - Babel & Terser 插件优化
- **适用场景**
  - JS 库打包 (React/Vue 组件库)
  - ES 模块化支持
  - 代码优化 (自动去除未使用代码)

#### **6.1.4 PNPM vs NPM vs Yarn**

- **NPM**
  - 传统安装机制 (node_modules 扁平化)
  - `package-lock.json` 版本锁定
- **Yarn**
  - PnP (Plug’n’Play) 无需 node_modules
  - Workspaces 依赖管理
- **PNPM**
  - `store` 共享依赖库 (避免重复安装)
  - Monorepo 最佳实践 (Turbo, Nx)

### **6.2 CI/CD 持续集成与部署**

- **GitHub Actions / Jenkins / GitLab CI**
  - 自动化构建、测试、发布
  - 版本管理与回滚
- **Docker & Kubernetes**
  - 容器化部署 (Dockerfile + Multi-stage Build)

### **6.3 代码质量与规范**
