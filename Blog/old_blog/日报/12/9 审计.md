| **`yarn why`**      | `yarn why react-server-dom-webpack`    | **查询依赖来源：** 解释为什么您的项目中安装了某个特定的包（例如 `react-server-dom-webpack`）。它能显示出引入该包的完整依赖链。 | 人可读的文本输出。         | N/A                                                          |
| **`yarn audit`**    | `yarn audit --level moderate`          | **安全审计：** 扫描项目所有依赖，检查是否存在已知的安全漏洞。 | 人可读的表格或摘要。       | **`--level moderate`**: 只报告中等（Moderate）及以上（High, Critical）严重程度的漏洞。 |
| **`yarn audit`**    | `yarn audit --json`                    | **安全审计（JSON 格式）：** 执行安全扫描，但将结果输出为结构化的 JSON 数据流。这主要用于自动化脚本和 CI/CD 集成。 | JSON 数据流。              | **`--json`**: 将结果格式化为机器可读的 JSON 格式。           |
| **`yarn list`**     | `yarn list --pattern react-server-dom` | **搜索依赖：** 在完整的依赖树中，只显示名称匹配特定模式（例如包含 `react-server-dom`）的包及其版本。 | 树状结构的人可读文本输出。 | **`--pattern`**: 匹配你想要搜索的包名称。                    |
| **`yarn outdated`** | `yarn outdated next react react-dom`   | **检查版本更新：** 检查指定的一个或多个依赖包是否有新的版本发布，并列出当前版本、期望版本和最新版本。 | 表格形式的人可读文本输出。 | 后面跟的包名是你想检查更新的特定依赖。                       |
